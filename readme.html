YouTube Live Chat Combiner

This Node.js application combines YouTube live chat and YouTube Shorts live chat into a single feed. It allows users to view both chats in one interface, making it easier to manage and interact with viewers across both platforms.

Features

  • Combines live chats from YouTube live streams and YouTube Shorts.
  • Displays the newest messages at the top of the chat feed.
  • Provides an interface to dynamically input and update the video IDs for both streams.

Prerequisites

  • Node.js installed on your machine.
  • A YouTube Data API v3 key.

Getting Started

Installation

  1. Clone the repository: 
    git clone <repository-url>
cd youtube-live-chat-combiner
  2. Install the dependencies: 
    npm install
  3. Set up your environment variables:

    Create a .env file in the root directory of the project (if it doesn't already exist) and add your YouTube Data API key:

    YOUTUBE_API_KEY=your_youtube_api_key_here

Running the Application

  1. Start the server: 
    node server.js
  2. Access the application:

    Open your browser and navigate to http://localhost:3000. You'll be able to see the combined chat feed for your YouTube live and Shorts live streams.

Configuring the Video IDs

The application allows you to dynamically input the video IDs for the live streams and Shorts directly through the interface.

Once you input the video IDs, click the Update button to refresh the chat feed with the new streams.

Obtaining a YouTube Data API Key

To use the YouTube Data API, you'll need an API key. Here's how to get one:

  1. Go to the Google Cloud Console: Visit Google Cloud Console.
  2. Create a new project:
    • Click on the project drop-down at the top of the page and select "New Project."
    • Name your project and click "Create."
  3. Enable the YouTube Data API v3:
    • Go to the APIs & Services section.
    • Click on "Enable APIs and Services" and search for "YouTube Data API v3."
    • Click on it and then click "Enable."
  4. Create credentials:
    • Go to the Credentials section.
    • Click "Create Credentials" and select "API Key."
    • Your new API key will be displayed. Copy it and paste it into your .env file as shown above.
  5. Restrict your API key (optional but recommended):
    • Click on the API key in the credentials list to edit it.
    • Under "API restrictions," select "YouTube Data API v3" to restrict your key's usage to the YouTube API only.
    • Under "Application restrictions," you can restrict it by IP address or HTTP referrers, depending on your needs.
